### Claude Code 创始人独家分享 10 条神级技巧！

>> 1、并行多开。同时开 3~5 个 git 分支，每个跑一个 Claude 会话，互不干扰。还可以设置快捷键一键切换

>> 2、复杂任务一定要先进计划模式，把心思花在打磨计划上，计划到位了 Claude 基本能一次性搞定。一旦感觉方向不对，别硬推，立刻切回计划模式重新来

>> 3.好好维护 CLAUDE.md 文件。每次纠正 Claude 的错误，顺手加一句 “把这条更新到 CLAUDE.md，下次别再犯”，时间久了出错率会明显下降

>> 4、把重复的事情封装成技能或斜杠命令，用 Git 管理，跨项目复用。

>> 5.让 Claude 自己修 Bug。比如用了 Slack MCP 之后，把 Bug 讨论帖往 Claude 一丢，说句 "fix" 就行。它的排查能力比你想象的强

>> 6.提示词可以更有创意。比如让 Claude 当考官：“先考考我这些改动理解对不对，我过了再提 PR”，或者说 “证明给我看这个能用”，让它对比 main 和功能分支的差异。修复完问题后觉得一般，就说 “根据你现在掌握的所有信息，推翻重来，给我个更优雅的方案

>> 7.选择合适的终端环境。比如 Ghostty，渲染和颜色支持都不错。可以用 /statusline 定制状态栏显示上下文用量和当前分支，给终端标签上色命名，一个标签对应一个任务。推荐试试语音输入，因为说话比打字快 3 倍，提示词会写得更详细

>> 8.使用子代理 Subagents。想让 Claude 更快完成任务的时候，请求后面加一句 "use subagents"，把独立的小任务丢给子代理处理，主代理的上下文就能保持干净

>> 9.利用 Claude 做数据分析。让它用 bq 命令行直接查 BigQuery，实时拉数据做分析。任何有命令行或 API 的数据库都能这么玩

>> 10.靠 Claude 学习新东西。在 /config 里开启解释型输出风格，让它讲清楚每个改动背后的原因。还可以让它生成 HTML 幻灯片讲解陌生代码，或者画 ASCII 图解释协议和架构
